# R4G1 Wire Format — RFC (DRAFT)

Status: DRAFT, Phase 0 deliverable of `docs/r4_graph_compiler_implementation_plan.md` (§5).
Implements PDF §18 (packed artifact format), §21 (key types), and Assumptions A1/A4 of the proof
model. Exact packed-record widths (PackedNode etc.) are chosen from measured graph sizes in
Phase 1 and frozen in the first numbered version of this document. Header-level width questions
are resolved (§9).

Terminology: `docs/transformerless/GLOSSARY.md`. Predecessor formats: TLA3/TLA4 (artifact),
TLS1 (store) — remain readable forever; R4G1 succeeds them.

## 1. Design rules (normative)

1. Single versioned container, magic `R4G1`. Explicit little-endian encoding, fixed integer
   widths, declared alignment, declared checksums.
2. Unknown **mandatory** sections MUST cause rejection. Unknown **optional** sections MAY be
   skipped per version policy.
3. All offsets are relative to their containing section start and validated with checked
   arithmetic before a `GraphView` is created.
4. Domain values crossing the serialization boundary use fixed-width types (`u8/u16/u32/u64/i32`),
   **never `usize`**, never platform-dependent layout.
5. The deployed view borrows immutable slices from caller-owned or memory-mapped bytes. The parser
   MUST NOT deserialize the graph into heap-resident object structures.
6. The parser rejects: out-of-range offsets, overlapping invalid ranges, non-canonical ordering,
   unsupported feature bits, invalid bytecode, and any bound declaration inconsistent with the
   actual ranges.
7. Canonical serialization is deterministic: identical graph content + format version ⇒ identical
   bytes (Gate E, semantics per decision D2).

## 2. Container layout

```
offset  size  field
0       4     magic "R4G1"
4       u16   format_version (major, minor as 2×u8)
6       u8    endianness marker (0x01 = little)
7       u8    alignment_log2 (section alignment, ≥ 3)
8       u64   total_len (bytes, includes header)
16      u32   section_count
20      u32   flags (feature bits; unknown mandatory bits ⇒ reject)
24      32B   artifact_cid (blake3 of everything after this field, zeroed during hashing)
56      32B   head_cid (blake3 of HEAD section body)
88      ..    section table, section_count entries × 16 bytes:
              { u32 section_id, u32 flags, u32 offset, u32 length }   (resolved, §9.1)
```

Section table entries are sorted by `section_id` (canonical ordering). Sections are padded to
`1 << alignment_log2`. Cache-line-sensitive sections (NODE/EDGE/ROUT/EMIT hot ranges) use 64-byte
alignment (Phase 7 may subdivide hot/cold).

Offsets and lengths are `u32`, section-relative, ceiling 4 GiB per section — ample against a
≤64 MB graph target and cheaper to read than u64 (§9.1). Remaining **TBD-w** (packed-record
widths: PackedNode, PackedEdge, RoutingOp, EmissionEntry) are frozen in **Phase 1** from measured
graph-size distributions (plan §5 Phase 1; PDF §21). Versioning gate for this freeze is
deterministic: pre-freeze artifacts MUST use `format_version.major = 0` (draft line), and stable
R4G1 starts at `major >= 1` only after widths are frozen. Readers/writers in the draft line MUST
reject any pre-freeze artifact that claims `major >= 1`.

## 3. Section inventory

| ID | Name | Mandatory | Contents |
|---|---|---|---|
| 0x01 | HEAD | yes | identities and limits (§4) |
| 0x02 | CODE | yes | token codes, rolling-state update programs, semantic-code layout |
| 0x03 | NODE | yes | packed region records (`PackedNode` array) |
| 0x04 | EDGE | yes | refinement (E_r), overlap (E_o), forward (E_f) edges + reverse index (E_b) |
| 0x05 | ROUT | yes | decision programs (DAG bytecode), prototypes, masks, shortlist tables |
| 0x06 | EMIT | yes | root token priors B(v), emission residuals ΔE, transition residuals ΔT |
| 0x07 | EXCT | opt. | exact-context residual evidence (successor of TLS1 store) |
| 0x08 | PROV | yes | provenance roots: source model κ, corpus CIDs, license refs, evidence roots |
| 0x09 | CERT | opt. | certification metadata: certificate CID(s), thresholds, metric refs |
| 0x0A | PTCH | opt. | patch-epoch header when the artifact is a patch layer (Phase 9) |
| 0x0B | SECT | opt. | per-section hash table — **reserved, future exploration** (§9.2) |

## 4. HEAD contents (normative field set, order fixed)

Identities:
- teacher model CID + pinned HF revision (40-hex), tokenizer identity CID
- compiler name + version, target architecture profile, deterministic seed
- corpus root CIDs (construction + held-out partitions, per decision D3)

Bounded-work constants (Theorem 4/9; validated against actual ranges):
- `A` max frontier width (u16), `C` max candidates per active node (u16),
- `W` signature words per region (u16), `E` max emission entries per region (u32),
- `K` token shortlist size (u16), `D` max decision-program steps (u32)

Starting defaults (freeze after Phase-2 frontier-width and emission histograms — issues #14/#15;
HEAD declares per-artifact maxima so values evolve per-compile without a format change):
`W=8` u64 words (512-bit codes), `A=32`, `C=16`, `E=64`, `K=8`, `D=64`, rolling window 8 tokens.

Graph dimensions: node_count (u32), edge_count (u32), depth_count (u8), vocab_size (u32),
signature_bytes (u16), ScoreQ semantic format declaration (i32 Q16.16 — §9.3).

Fallback policy: per-`ResolutionStatus` behavior codes (continue / widen / consult EXCT /
certified fallback / abstain) — the D4 policy, plus widening and exhaustive-fallback bounds.

Compatibility: min runtime version, feature bits actually required, epoch/patch-chain info
(parent CID, layer index, max layers) when PTCH present.

### 4.1 v0 draft-line HEAD payload (implemented)

Fixed 224-byte little-endian prefix, implemented in `uor-r4-graph-format` (`Head`). Draft-line
readers require the payload to be exactly 224 bytes: trailing bytes are rejected (not ignored)
so a future HEAD extension must arrive with a minor-version bump (§8).

```
offset  size  field
0       32B   teacher_cid
32      32B   tokenizer_cid
64      32B   corpus_construction_cid
96      32B   corpus_certification_cid
128     20B   hf_revision (ASCII hex, zero-padded right; opaque to the v0 validator)
148     32B   compiler_version_cid
180     u16   A (max frontier width)
182     u16   C (max candidates per active node)
184     u16   W (signature words per region)
186     u16   K (token shortlist size)
188     u32   E (max emission entries per region)
192     u32   D (max decision-program steps)
196     u32   node_count
200     u32   edge_count
204     u8    depth_count
205     5B    fallback policy codes (one per ResolutionStatus: Supported, Boundary,
              BackedOff, Novel, Contradictory — the D4 policy)
210     2B    reserved (0)
212     u16   signature_bytes (byte-exact signature width over word-aligned
              storage: `(W-1)*8 < signature_bytes <= W*8`; stage-2 cross-check.
              The padding bytes between the signature and each region's W-word
              prototype/mask extent must be zero — validated for every node
              whose prototype/mask ranges resolve)
214     u16   min_runtime_major
216     u16   min_runtime_minor
218     u16   feature_bits_required
220     u32   vocab_size
```

The epoch/patch-chain compatibility fields and the widening/exhaustive-fallback bounds remain
unassigned in v0 (PTCH-era, Phase 9); the draft line carries only the fixed prefix above.

## 5. Section payload sketches

- **CODE**: token→code table (fixed-width entries), rolling-state program bytecode
  (shift/rotate/xor/add op list, validated), semantic-code word layout (W words × u64).
- **NODE**: `PackedNode[]` per PDF §21 — v0 draft-line layout, implemented: 30-byte records,
  contiguous, no padding, decoded on demand. Record count must equal HEAD `node_count`.
  ```
  offset  size  field
  0       u32   child_start          (range over the canonical edge array)
  4       u16   child_len            (≤ HEAD A)
  6       u32   forward_start        (range over the EDGE reverse index)
  10      u16   forward_len
  12      u32   emission_start       (byte range over the EMIT remainder)
  16      u16   emission_len         (≤ HEAD E)
  18      u32   prototype_word_start (u64-word index into ROUT)
  22      u32   mask_word_start      (u64-word index into ROUT)
  26      u16   radius
  28      u8    depth                (< HEAD depth_count)
  29      u8    flags
  ```
- **EDGE**: v0 draft-line layout, implemented: the canonical edge array — 16-byte records
  `(u32 src, u32 dst, i32 score_q, u8 kind, u8 flags, u16 reserved)` — with stable edge IDs =
  array index, immediately followed by the reverse index: `edge_count` × u32 edge IDs, ranges
  over which are given by the PackedNode forward fields (sorted by target, Theorem 7). Section
  length must equal `edge_count × 20` exactly. v0 reverse-index validation is the Theorem-7
  *existence* + per-node wiring check: every reverse entry < `edge_count`, every required edge
  kind appears in the reverse index, and every node's declared reverse range resolves only to
  edges whose `dst` equals that node.
  Edge-kind discriminants are stable: `0=refinement/abstraction`, `1=similarity`,
  `2=predictive-forward`, `3=causal`, `4=temporal`, `5=constraint`, `6=goal-progress`,
  `7=evidence/provenance`, `8=evidential-predecessor`; unknown kinds are rejected unless
  `kind & 0x80 != 0` (optional-kind space). `feature_bits_required` bit 0
  (`FEATURE_EDGE_ALGEBRA_V1`) enables contribution-id semantics in `reserved` for evidence-bearing
  kinds and canonical edge ordering by `(src, kind, dst, contribution_id)`.
- **ROUT**: v0 draft-line layout, implemented: a single decision program followed by an
  optional trailing shortlist table (prototype/mask words are referenced into the section by
  u64-word index; a typed region layout is a later slice). Fixed-width ops (1-byte opcode +
  little-endian operands):
  ```
  opcode  size  operands
  0x00    1     HALT — program terminator; trailing bytes form the shortlist table
  0x01    12    TEST_POPCOUNT_LE { word u8, mask u64, threshold u16 }
  0x02    3     JMP_FWD { delta_ops u16 } — target op index = current + 1 + delta_ops
  0x03    7     LEAF { shortlist_start u32, shortlist_len u16 } — byte range into the table
  ```
  Bytecode validated per §6 item 6: known opcodes; operands in range (`word < HEAD.W`,
  `threshold ≤ 64`, shortlist ranges within the trailing table — with no table present,
  `shortlist_len` must be 0); the program ends at the first HALT, or at section end with a
  final LEAF; jumps are forward-only by construction (unsigned forward delta ⇒ acyclic) and
  in-bounds; static op count ≤ HEAD.D (bounds every execution path under forward-only jumps).
- **EMIT**: root prior block B(v) + residual lists keyed by region (sparse, bounded ≤ HEAD.E per
  region), shared emission blocks by reference. Each table carries a dyadic **storage
  descriptor** — v0 draft-line layout, implemented: a 4-byte prefix `{width u8 (0=i8, 1=i16,
  2=i32), shift i8, zero_point i16}` per section, validated (width in set, `|shift| ≤ 31`);
  entries decode to semantic ScoreQ by shift+add at table-read time — no multiply (§9.3). The
  remainder of the section is opaque to the v0 validator; node emission ranges resolve as byte
  ranges over it.
- **EXCT**: exact-context residual evidence, successor of the TLS1 graded store. Two body
  forms are defined; both carry the validated 4-byte storage-descriptor prefix.
  **RX1 (deployed form, 2026-07-22)**: magic `RX1\0` | `levels u8` (= STAGES+1) | `pad 3` |
  per level: `key_count u32`, then per context: `key_len u8` | key bytes (class-code prefix,
  B-tree orderable, preserving prefix backoff probing) | `total u32` | `entry_count u32` |
  entries `(token u32, residual i32)` ascending token. Residuals are baked at compile time:
  `ΔX = ScoreQ::from_logprob(ln((count+1)/(total+V))) − B(v)` (add-one smoothing), top-X by
  (count desc, token asc). The deployed read path is integer-only; parity with the probe-time
  quantized reference is machine-tested (`tests/score.rs::rx1_baked_residuals_match_probe_time_
  quantization`). **Legacy TLS1 (certifier-side carryover)**: raw TLS1 store bytes; the
  reference scorer quantizes at probe time inside one delimited compiler-side float block
  (never reached by the deployed RX1 path). A reader distinguishes forms by attempting the RX1
  parse first (magic + exact consumption), then the TLS1 parsers.
- **PROV**: source/corpus/observation CIDs, evidence roots for deletion support, license
  identifiers (SPDX) for redistributable corpora (D3).
- **CERT**: certificate CID(s) + metric/threshold summary; the certificate document itself lives
  in the object store, not inline.

## 6. Validation invariants (Assumption A1 / Theorem 8)

Stage 1 — structural (bytes → section table):
1. magic, version supported, endianness marker, total_len == actual, section table in bounds,
   sorted, non-overlapping, aligned;
2. unknown mandatory section or feature bit ⇒ reject;
3. all offsets/lengths checked with `checked_*` arithmetic.

Stage 2 — semantic (sections → `GraphView`):
4. every packed range (children/forward/emission/prototype/mask) resolves within its target
   section — prototype/mask ranges including their full W-word extent within ROUT, with the
   padding bytes between `signature_bytes` and each W-word extent required to be zero
   (§4.1 word-aligned signature storage);
5. every edge src/dst < node_count; child ranges resolve to refinement edges from the containing
   node; reverse-index entries resolve to canonical edges whose target equals the containing
   range's node (Theorem 7);
6. ROUT bytecode: known opcodes, operands in range, acyclic, depth ≤ HEAD.D;
7. HEAD constants ≥ actual maxima observed in sections (frontier/candidates/emission bounds are
   honest);
8. emission/EXCT keys canonically ordered; ScoreQ storage descriptors valid (width in set, shift
   within declared bounds);
9. CID fields recompute to the artifact bytes (integrity).

Only after both stages may a `GraphView` be constructed; runtime table access then iterates
validated ranges only.

Phase mapping (normative intent):
- plan §5 Phase 0: publish this RFC draft and structural/semantic invariant set.
- plan §5 Phase 1: implement two-stage validator + typed `GraphView<'a>` and freeze packed widths.
- plan §6 obligations: this section operationalizes **A1** (validated-only `GraphView`) and **T8**
  (artifact-validation soundness via checked ranges → typed views).

## 7. Reproducibility semantics (decision D2)

Normative definitions for **byte reproducibility** (BR), **behavioral equivalence** (BE, PDF §15),
and the confidence-bounded empirical-claim protocol are specified in
`docs/transformerless/EQUIVALENCE_AND_EMPIRICAL_PROTOCOL.md`.

- Certificate-bearing artifacts MUST satisfy BR in canonical deterministic mode (Gate E).
- Platform-accelerated local compiles may use BE validation under the protocol above.
- The mode used is recorded in HEAD (compiler version + flags) and in CERT.

## 8. Versioning policy

- `format_version.major` bump: any mandatory-section or semantic change. Readers reject newer
  majors.
- `minor` bump: new optional sections/feature bits. Old readers skip unknown optional content.
- One active major at a time in this repository; TLA3/TLS1 readers remain as legacy fixtures.

## 9. Resolutions (2026-07-21) and remaining open questions

### Resolved

1. **Section-table entry and offset widths.** 16-byte entries `{u32 id, u32 flags, u32 offset,
   u32 length}`; u32 section-relative offsets/lengths (≤4 GiB per section vs ≤64 MB target).
   Rationale: fewer bytes read (Gate D) with ample headroom. Accepted.
2. **Checksums.** Single artifact CID + HEAD CID for now. **Future exploration (reserved section
   0x0B SECT):** a per-section hash table enabling mmap partial verification — verify or hot-swap
   one section without reading the whole artifact; candidate for a v1.x feature bit or the
   PTCH-era format. Revisit when patch layers (Phase 9) or partial-memory deployments make it
   concrete.
3. **ScoreQ.** Two-tier design. *Semantic*: one global format — i32 Q16.16 declared in HEAD
   (range/resolution ample for log-domain scores; a single declared semantics keeps A4 and
   Theorem 6 replay simple). *Storage*: per-table dyadic descriptors `{width, shift, zero_point}`
   (see EMIT, §5); decode is shift+add at table-read time — mul-free, matching the existing
   decode-on-demand pattern (`runtime.rs:134-147`, `compiler.rs:456-470`). Per-depth scales fall
   out as a special case. Per-table parameters chosen from Phase-4 residual histograms.
4. **v0 draft-line packed layouts (2026-07-22).** The HEAD 224-byte prefix (§4.1), the 30-byte
   PackedNode, the 16-byte PackedEdge + reverse index, the ROUT v0 opcode set, and the 4-byte
   EMIT/EXCT storage descriptor (§5) are adopted for the draft line (`major = 0`) and
   implemented with stage-2 semantic validation in `uor-r4-graph-format`. They remain subject
   to confirmation or adjustment at the Phase-1 width freeze from measured graph sizes; only
   then does a stable `major >= 1` start. Two v0 approximations are called out: the Theorem-7
   reverse-index check is *existence*-based (every reverse entry resolves; every canonical edge
   has at least one reverse entry — O(edge_count²) worst case at validation time), with full
   per-node reverse-range wiring deferred; and node emission ranges resolve as byte ranges over
   the EMIT remainder until the EMIT table layout lands. A container without a HEAD section
   stays stage-1-only (bootstrap fixtures, pure section carriers).
5. **Phase-4 EMIT table layout and EXCT key format (2026-07-22).** The EMIT remainder layout
   landed with the Phase-4 scored-graph artifacts (`uor-r4-core::transformerless::score`): a
   16-byte root-prior block header `{u32 entry_count, u32 total_count, i32 floor_score_q,
   u32 reserved}` followed by `entry_count` × `(i32 token, i32 score_q)` pairs in ascending
   token order, then the per-region residual lists, each `emission_len` × `(i32 token,
   i32 score_q)` pairs in ascending token order. Node `emission_start` is a byte offset into
   the remainder and `emission_len` is an **entry count** (≤ HEAD E); the stage-2 range check
   stays the v0 byte-range approximation (conservative: `start + entries ≤ start + 8·entries
   ≤ remainder` for honest writers), with entry-typed resolution a format-crate slice of the
   Phase-1 width freeze. EXCT keeps the prefix-backoff TLS1 carryover (descriptor + raw TLS1
   bytes) as compiler input. The scored artifact emits RX1 residuals (baked per-key
   `ScoreQ` values, removing probe-time float quantization) with the same deepest-prefix
   lookup semantics; raw TLS1 remains a legacy compatibility input. EDGE kind tags:
   0 = refinement/abstraction, 1 = similarity, 2 = predictive-forward, 3 = causal,
   4 = temporal, 5 = constraint, 6 = goal-progress, 7 = evidence/provenance,
   8 = evidential-predecessor.
